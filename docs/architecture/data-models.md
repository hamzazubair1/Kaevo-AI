# Kaevo AI — Data Models

> Entity relationship diagrams and database schemas for the unified CRM, Analytics, and Agent state.

---

## Architecture Overview

Kaevo AI uses a unified PostgreSQL database (via Supabase) serving both System 1 (Inbound) and System 2 (Outbound).

```
┌────────────────────────────────────────────────────────┐
│                   Unified Database                     │
├──────────────┬──────────────┬─────────────┬────────────┤
│   Accounts   │    CRM       │   Agents    │ Analytics  │
│   (Auth)     │   (Sales)    │  (System)   │ (Metrics)  │
└──────────────┴──────────────┴─────────────┴────────────┘
```

## Core CRM Schema

### `businesses`
Central entity for both inbound clients and outbound prospects.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PK | Unique identifier |
| `name` | String | Not Null | Company name |
| `website` | String | Unique | Company domain/URL |
| `industry` | String | | Target industry categorization |
| `location` | String | | Physical location / HQ |
| `size_estimate`| String | | Employee count range |
| `source` | String | | e.g., 'inbound_chat', 'outbound_discovery' |
| `status` | Enum | | `discovered`, `analyzed`, `prospect`, `client`, `archived` |
| `created_at` | Timestamp| Auto | Record creation |
| `updated_at` | Timestamp| Auto | Last update |

### `contacts`
Individuals associated with a business.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PK | Unique identifier |
| `business_id`| UUID | FK (businesses)| Associated company |
| `first_name` | String | | |
| `last_name` | String | | |
| `email` | String | Unique | Primary contact email |
| `phone` | String | | |
| `role` | String | | Job title / position |
| `is_primary` | Boolean| Default false | Primary contact for the account |

### `interactions`
Log of all communications (chat, email, meetings).

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PK | Unique identifier |
| `business_id`| UUID | FK (businesses)| |
| `contact_id` | UUID | FK (contacts) | Optional |
| `agent_id` | String | | ID of AI agent (if AI interaction) |
| `type` | Enum | Not Null | `chat`, `email_sent`, `email_reply`, `meeting` |
| `content` | Text | | Transcript or email body |
| `timestamp` | Timestamp| Default NOW()| When it occurred |

## System 2 (Outbound) Specific Schema

### `website_analyses`
Results from Agent B2.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PK | |
| `business_id`| UUID | FK (businesses)| |
| `tech_stack` | JSONB | | Detected technologies (CMS, tools) |
| `gaps` | JSONB | | Array of identified automation gaps |
| `gap_count_critical` | Integer | | |
| `analyzed_at`| Timestamp| | |

### `opportunity_scores`
Results from Agent B3.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `business_id`| UUID | PK, FK | 1:1 with business |
| `total_score`| Float | 0-100 | Composite score |
| `dimensions` | JSONB | | Breakdown of score by category |
| `classification`| Enum | | `hot`, `strong`, `moderate`, `low`, `none` |
| `scored_at` | Timestamp| | |

## Sales & Proposals Schema

### `deals` (Pipeline)
Tracks revenue opportunities.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PK | |
| `business_id`| UUID | FK (businesses)| |
| `service` | String | | Primary service offered |
| `amount` | Decimal| | Projected value |
| `stage` | Enum | | `qualified`, `meeting_booked`, `proposal_sent`, `won`, `lost` |
| `probability`| Integer| 0-100 | Likelihood to close |
| `close_date` | Date | | Expected close |

### `proposals`
Generated by Agents A3 or B4.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PK | |
| `deal_id` | UUID | FK (deals) | |
| `version` | Integer| Default 1 | |
| `content_url`| String | | Link to generated PDF/HTML in storage |
| `status` | Enum | | `draft`, `pending_review`, `sent`, `viewed`, `accepted`, `rejected` |
| `created_at` | Timestamp| | |

## Agent Configuration Schema

### `agent_configs`

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `agent_id` | String | PK | e.g., 'agent-a1-receptionist' |
| `version` | String | | Semver |
| `system` | String | | 'system-1' or 'system-2' |
| `llm_config` | JSONB | | Provider, model, temperature settings |
| `prompts` | JSONB | | System prompts and templates |
| `is_active` | Boolean| Default true | |
