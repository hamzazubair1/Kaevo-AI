# Standard Operating Procedures (SOPs)

> Step-by-step guides for all repeatable processes at Kaevo AI.

## Structure

```
sops/
├── engineering/         # Engineering and development SOPs
├── client-management/   # Client-facing process SOPs
├── onboarding/          # Team and client onboarding SOPs
├── deployment/          # Deployment and release SOPs
└── README.md
```

## Documentation

- See [SOPs Documentation](../docs/sops.md) for the complete SOP index and template
- Every SOP must follow the standard template
- SOPs are reviewed quarterly

## Rules

1. If you do it twice, write an SOP
2. Write for someone who has never done it before
3. Include screenshots and examples
4. Assign an owner to every SOP
5. Keep SOPs updated when processes change
